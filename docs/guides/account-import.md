# 压缩包导入（推荐）

支持批量导入账号压缩包：**每个账号一个独立子文件夹**，文件夹内包含一个 `.json` + 一个 `.session`。

## 批量导入压缩包结构（示例）

例如你要导入 2 个账号：

```
accounts.zip
  ├─ 8613111111111
  │   ├─ 8613111111111.json
  │   └─ 8613111111111.session
  └─ 8615119714541
      ├─ 8615119714541.json
      └─ 8615119714541.session
```

### 命名规则

- 子文件夹名建议使用手机号（或你能区分账号的唯一标识）。
- `.json` 与 `.session` 文件名建议与子文件夹同名（上面的例子就是同名）。
- 每个子文件夹内只要能找到 **1 个 `.json` + 1 个 `.session`** 即可。

## Docker 部署下导入的文件会存哪？

如果你使用 Docker，所有 session 会写入：

- `./docker-data/sessions/`

不要手工乱改里面的文件名，避免程序找不到 session。
